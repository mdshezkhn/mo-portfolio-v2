# Sprint 0 Execution Plan – Content Extraction

**Version:** 1.0 (Approved)

Sprint 0 is not "writing content" – it is **extracting the professional story**. One session per task, sequential, card-by-card. Format: user provides → Duo produces → user reviews → acceptance criteria. No HTML until exit criteria met.

## Task 0.0 – Proof Folder & Asset Register (prerequisite)
User assembles Proof Folder **locally/privately – never committed to this repository** (CV, certificates, employment letters, PGCE work, photos, metrics). Duo produces ASSETS.md register. Accept: register exists, all held assets logged, gaps explicit.

## Task 0A – Hero
User: title, location, availability, headshot candidates, CV PDF. Duo: 3 value proposition options (facts only), availability wording, CTA text. Accept: all Must slots filled; headshot Verified.

## Task 0B – My Story
Duo runs interview (origin, why education, why international, why China, what drives you now) → 2 drafts <=250 words. Accept: approved verbatim; every fact traceable to answers.

## Task 0C – Teaching Impact (card-by-card)
Per card: user describes real example (what/where/when/evidence) → Duo drafts card → approve/revise → next. Accept: 3–6 approved cards, each citing proof or direct statement.

## Task 0D – Teaching Philosophy
User: beliefs + one real example per topic. Duo drafts 450–600 words, revised until it sounds like the user. Accept: <=700 words, topics anchored to real practice.

## Task 0E – Leadership
Cards as 0C (2–4, LEADER). Principles: user states raw, Duo only tightens wording, never originates. Accept: >=2 sourced cards, 3–4 genuine principles.

## Task 0F – Research
User: PGCE topic, findings in own words, one concrete classroom change. Duo: <=300-word principal-friendly draft with classroom-change closing. Accept: no jargon, approved verbatim.

## Task 0G – Impact Highlights Verification
User: exact figures + verifying document per stat. Duo: strip content for stats passing verification + Scale/Reach/Improvement/Leadership test. Accept: >=3 verified → ships; fewer → hidden, no exceptions.

## Task 0H – Classroom Moments
User: photo candidates + per-image consent + caption facts. Duo: curation (8–20), captions, alt text, tags. Accept: every image passes safeguarding checklist, logged with consent status.

## Task 0I – Credentials & Contact
User: credentials vs certificates, languages, skills, Currently Learning, email, LinkedIn, closing line. Duo: formatted lists, safeguarding statement, email obfuscation plan. Accept: every credential matches proof.

## Dependencies
0.0 → 0A, 0B, 0F, 0H, 0I · 0B → 0D · 0C → 0E · 0C/0E → 0G

## Exit Criteria (gates Sprint 1)
1. All Must slots filled with approved, sourced content
2. Asset Register complete – nothing used without Verified status
3. Impact Highlights verified or formally hidden
4. Zero [NEEDS INPUT] markers in launch scope
5. Explicit user sign-off: "Sprint 0 complete"
